# Frontend Repo Rules

## Repo URLs

- Frontend repo: `https://github.com/FidelCoder/cadrisFE.git`
- Backend repo: `https://github.com/FidelCoder/cadrisBE.git`

## Working rules

- Keep this repo focused on the Next.js PWA and client runtime only.
- Route persistence, storage, and export-preview writes through the backend API.
- Cap markdown files in this repo at three: `README.md`, `EXECUTION_PLAN.md`, and `REPO_RULES.md`.
- Keep UI primitives lightweight and consistent with the dark, premium camera context.
- Favor modular TypeScript over quick inline logic for recorder, vision, and planner code.
- Preserve architecture so face tracking and speaker detection can be upgraded later without rewriting screens.
